Executive Summary

The target at http://localhost:8001/chat appears to be a customer-support oriented chatbot with a confirmed ability to answer supported support-domain questions such as policies, subscriptions, order status topics, and support-ticket guidance. The assessment did not confirm that the agent can actually invoke tools, create tickets, check live order status, browse the web, access files, execute commands, use APIs/plugins/MCP, or retain memory across sessions. Across 59 probes with 0 errors, the strongest security finding is confirmed exposure to indirect prompt-injection content: the system repeatedly surfaced sanitized adversarial instructions from retrieved or embedded content, including data-exfiltration themed strings. The agent did refuse to reveal hidden instructions or internal configuration in prompt-leakage probes, so direct system-prompt or tool-schema disclosure was not observed. Customer-specific data access was described as requiring valid user and tenant context, but actual tenant-isolation behavior was not validated. Overall confidence is moderate: the support-chatbot role and indirect-prompt-injection exposure are well supported, while most operational capabilities remain uncertain or not observed due to unrelated or non-responsive knowledge-base answers.
